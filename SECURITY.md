# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in Poof Backend, please report it privately and responsibly:

1. **Do not** open a public GitHub issue with exploit details.
2. Send a report to the maintainers via private channel/email with:
   - vulnerability description,
   - impact and affected components,
   - steps to reproduce,
   - optional proof-of-concept.
3. Include your contact details for follow-up.

We will acknowledge reports as quickly as possible, validate the issue, and coordinate a fix and disclosure timeline.

## Disclosure Guidelines

- Please give maintainers reasonable time to patch before public disclosure.
- We will publish remediation notes once the fix is released.
