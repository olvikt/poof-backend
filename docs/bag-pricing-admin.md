# Bag pricing in admin panel

## Где хранятся тарифы

Тарифы по количеству мешков хранятся в таблице `bag_pricings`.

Поля:
- `bags_count` — количество мешков (уникально);
- `price` — стоимость в грн (целое, неотрицательное);
- `is_active` — активен ли тариф для новых заказов;
- `sort_order` — порядок отображения на клиентском экране.

## Как админ управляет тарифами

В Filament добавлен ресурс **"Тарифи мішків"**:
- можно создавать новый вариант количества мешков;
- можно редактировать цену, активность и порядок;
- дубли `bags_count` запрещены (уникальный индекс + валидация формы).

## Как теперь считается цена

- Клиентский экран заказа получает `pricing` только из активных записей `bag_pricings`.
- Цена sticky-summary считается через `Order::calcPriceByBags()`, которая читает тариф из БД.
- Если выбранный `bags_count` отсутствует среди активных тарифов, оформление заказа блокируется валидацией (без небезопасного fallback).

## Миграция и начальные данные

Миграция `2026_03_31_200000_create_bag_pricings_table.php`:
- создаёт таблицу `bag_pricings`;
- сразу добавляет дефолтные тарифы:
  - `1 => 40`
  - `2 => 55`
  - `3 => 70`

После миграции система продолжает работать без ручной настройки.

## Влияние на старые заказы

Существующие заказы не переоцениваются:
- историческая стоимость уже хранится в `orders.price`;
- новые тарифы применяются только при расчёте цены новых заказов.

## Возможные риски

- Если админ деактивирует все тарифы, новый заказ оформить нельзя до включения хотя бы одного тарифа.
- При изменении цен в админке обновлённые тарифы применяются сразу для новых оформлений.
